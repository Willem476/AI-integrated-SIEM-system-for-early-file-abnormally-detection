# Endpoint Telemetry Setup: Implementing Microsoft Sysmon 

An **Endpoint** (like Windows Server 2022 target) is where the actual cyber activities happen. To detect malicious files early using our Random Forest AI model, we need deep, granular visibility into system events. 

Standard Windows Event Logs are insufficient. Therefore, we deploy **Microsoft Sysmon** combined with a security-hardened configuration to log advanced events like process creations, network connections, and critical file modifications (`FileCreate`, `FileDelete`). 

---

## Step-by-Step Installation & Verification Guide

### Step 1: Download Sysmon
1. Navigate to the official Microsoft Sysinternals page: [Microsoft Sysmon Download](https://learn.microsoft.com/en-us/sysinternals/downloads/sysmon).
2. Download the Sysmon `.zip` package, extract it on your Windows Server endpoint, and move the extracted folder to a permanent path (e.g., `C:\Sysmon\`).

### Step 2: Download Olaf Hartong's Hardened Security Configuration
Default Sysmon logging generates massive noise. To streamline feature extraction for our Machine Learning model, we utilize the renowned MITRE ATT&CK mapped configuration by Olaf Hartong.
1. Download the modular configuration file directly from the official repository: [Olaf Hartong's Sysmon Config](https://github.com/olafhartong/sysmon-modular/blob/master/sysmonconfig.xml).
2. Save this `sysmonconfig.xml` file directly inside your `C:\Sysmon\` directory.

### Step 3: Install Sysmon via PowerShell
1. Click on the Start menu, search for **Windows PowerShell**, right-click it, and select **Run as Administrator**.
2. Navigate to your Sysmon directory by executing:
   ```powershell
   cd C:\Sysmon\
   ```
3. Run the installation command utilizing the downloaded XML configuration:
   ```powershell
   .\Sysmon64.exe -i .\sysmonconfig.xml
   ```
4. A Microsoft software license terms window will pop up. Click **Agree** to finalize the background service installation. Sysmon is now continuously monitoring and recording deep system events!

### Step 4: Verify Sysmon Status (Services & Event Viewer)
To ensure Sysmon is operating properly and collecting data, perform the following validation checks:

#### Check via Windows Services:
1. Press `Windows Key + R`, type `services.msc`, and press `Enter`.
2. Scroll down and locate the service named **System Monitor (Sysmon)** or **Sysmon64**.
3. Verify that the **Status** is listed as **`Running`** and the **Startup Type** is set to **`Automatic`**.

#### Check via Event Viewer:
1. Press `Windows Key + R`, type `eventvwr.msc`, and press `Enter`.
2. In the left console tree, navigate to the following path:
   ```text
   Applications and Services Logs -> Microsoft -> Windows -> Sysmon -> Operational
   ```
3. Click on **Operational**. You should immediately see a live stream of event logs generated by Sysmon. Look for crucial event IDs such as **Event ID 1 (Process Creation)** or **Event ID 11 (FileCreate)** to confirm that your endpoint telemetry is rich, active, and ready for ingestion!
